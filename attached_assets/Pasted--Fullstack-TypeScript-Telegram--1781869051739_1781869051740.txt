## Роль и Цель
Ты — опытный Fullstack TypeScript разработчик. Твоя задача — создать Telegram-бот для инвентаризации кофейни. Бот должен быть написан строго под стек среды Replit, обладать модульной архитектурой и поддерживать одновременную работу до 4 сотрудников без перезаписи данных друг друга.

Интерфейс должен быть максимально чистым, работать ИСКЛЮЧИТЕЛЬНО на inline-кнопках (через редактирование текущего сообщения `ctx.editMessageText`), чтобы не засорять чат.

## Технологический стек (Strict Stack)
- **Runtime:** Node.js 24, TypeScript 5.9
- **Структура:** pnpm workspaces (монорепо)
- **Web-сервер:** Express 5
- **Telegram Bot:** Telegraf (версия 4.x, режим Polling)
- **База данных:** PostgreSQL (Встроенная Replit DB)
- **ORM:** Drizzle ORM + drizzle-zod
- **Сборщик:** esbuild (бандлинг в ESM)
- **Логирование:** pino + pino-pretty

## Архитектура Базы Данных (Drizzle ORM)
Спроектируй схему со следующими таблицами:
1. **users:** `id` (Telegram ID), `username`, `lang` ('en' | 'sr'), `is_admin` (boolean).
2. **locations:** `id` (serial), `name_en`, `name_sr`. (Начальные данные: Vračar, Dorćol).
3. **categories:** `id` (serial), `name_en`, `name_sr`.
4. **products:** - `id` (serial), `category_id`, `name_en`, `name_sr`
   - `measurement_type`: enum ('numeric', 'color', 'both')
   - `unit`: string (например: pcs, pack, l, ml, kg) — используется, если тип numeric или both.
5. **active_sessions:** Учет сессий сотрудников. `id`, `user_id`, `location_id`, `status` ('in_progress', 'completed'), `updated_at`.
6. **inventory_records:** Текущие сохраненные ответы. `id`, `session_id`, `product_id`, `val_numeric` (real, nullable), `val_color` (enum: 'green', 'yellow', 'red', nullable).

## Логика работы и Пользовательский сценарий (User Flow)

### 1. Старт и Настройки (Язык и Локация)
- Команда `/start` проверяет пользователя в БД. Если языка нет, выводится inline-выбор: **English 🇬🇧 / Српски 🇷🇸**.
- Далее открывается выбор локации: **Vračar / Dorćol**.
- Все данные сессии привязываются к связке `Пользователь + Локация` в PostgreSQL. Логика не должна сбоить, если 2-3 сотрудника одновременно заполняют инвентаризацию на разных или одинаковых точках.

### 2. Главное меню категорий
- Отображает категории в 2 столбца (Inline-кнопки).
- Под категориями всегда закреплены 2 системные кнопки:
  - `📋 Pregled izveštaja / View Report` — просмотр текущего прогресса.
  - `🔄 Poništi izveštaj / Reset Report` — полная очистка текущей сессии (с inline-подтверждением "Вы уверены?").

### 3. Навигация внутри категории и Ввод данных
При клике на категорию открывается список её товаров.
- **Индикатор заполненности:** Если позиция уже заполнена в этой сессии, перед её названием на кнопке динамически отображается галочка `✅` (например, `✅ Obično mleko`).
- Внизу списка товаров всегда есть кнопки: `⬅️ Ka kategorijama / To Categories` и `📋 Pregled izveštaja`.

**При клике на товар бот проверяет `measurement_type`:**
- **Тип А (Numeric):** Бот меняет текст сообщения на "Unesite količinu / Enter quantity (unit):". Пользователь пишет число в чат. Бот с помощью Telegraf Hear/FSM ловит текстовое сообщение, валидирует (что это число), удаляет сообщение пользователя (чтобы не было мусора), обновляет данные в БД, отправляет уведомление "Dodato ✅" и **сразу же возвращает пользователя в меню текущей категории** (обновляя сообщение).
- **Тип Б (Color / Светофор):** Открывается inline-подменю с кнопками: 
  - `🟢 Zeleno (Mnogo / В наличии)`
  - `🟡 Žuto (Zadržava se / Хватит на пару дней)`
  - `🔴 Crveno (Hitno / Горим, заказать)`
  - `⬅️ Nazad / Back`
  При клике статус мгновенно пишется в БД, и бот возвращает юзера в меню категории.
- **Тип В (Both):** Сначала запрашивается ввод числа (текстом), а затем сразу открывается выбор цвета (inline-кнопками).

### 4. Исправление ошибок
Если пользователь ошибся, он может просто кликнуть на эту же позицию еще раз — новое значение перезапишет старое в `inventory_records`.

### 5. Просмотр и отправка отчета в Канал
При клике на `📋 Pregled izveštaja`:
- Формируется красивый текстовый шаблон: Имя сотрудника, Локация, Дата, и список позиций, разбитый по категориям. Числа выводятся с единицами измерения (`10 pcs`, `2 l`), цвета выводятся в виде эмодзи (`🟢`, `🟡`, `🔴`). Если тип `both` — выводятся и число, и цвет рядом.
- Под текстом доступны кнопки: 
  - `✅ Posalji / Submit` — закрывает сессию, отправляет этот отчет в админ-канал (ID канала берется из переменной окружения `TELEGRAM_CHANNEL_ID`). **Обязательно добавить динамический хэштег даты** (например, `#report_19_06_2026`) в конец сообщения для удобного поиска по архиву канала.
  - `✏️ Uredi stavku / Edit` (возврат к категориям).
  - `⬅️ Nazad / Back`.

## Админ-панель (Добавление позиций без кода)
Для пользователей со статусом `is_admin = true` доступна секретная команда `/admin` или кнопка управления.
Через пошаговый inline-интерфейс админ может:
1. **Добавить категорию:** ввести названия на английском и сербском.
2. **Добавить позицию:** - Выбрать существующую категорию из inline-списка.
   - Ввести названия (English / Srpski).
   - Выбрать тип исчисления через inline-кнопки: `Цвета`, `Веса/Количества`, `Веса + Цвета вместе`.
   - Если выбран вес/количество — бот просит ввести единицу измерения (шт, уп, л, мл, кг).
Все изменения сразу пишутся в БД и становятся доступны сотрудникам.
